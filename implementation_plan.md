# Implementation Plan

[Overview]
Refactor EPUBMaker into a modular feature architecture that cleanly separates UI concerns from EPUB processing, while also improving reliability, validation, and user feedback.

The current implementation in `src/app/tools/epub-maker/page.tsx` is a single large client component containing UI rendering, clipboard ingestion, HTML sanitization, image fetching/embedding, EPUB packaging, filename generation, and persistence logic. This concentration of concerns makes the page difficult to reason about, difficult to test in isolation, and expensive to evolve when adding new capabilities.

The refactor will split EPUBMaker into a feature module with explicit boundaries: (1) presentational UI components, (2) state orchestration via a dedicated hook/reducer, and (3) pure domain utilities for transformation/sanitization/building. This separation reduces cognitive load, enables focused unit tests for EPUB logic without rendering React, and allows UI iteration without risking packaging logic regressions.

In addition to structural refactoring, the plan includes broader functional improvements requested by the user: stricter input validation and duplicate handling, configurable sanitization behavior, improved image import diagnostics, explicit progress states during generation, chapter reorder/rename support, and more robust error reporting. These enhancements are designed to remain incremental and compatible with existing behavior while improving end-user control and maintainability.

[Types]
Introduce a dedicated type system for EPUBMaker domain entities and UI state, replacing ad-hoc local types in page scope.

Define the following in `src/features/epub-maker/types.ts`:

- `export type PageInputKind = "html" | "text";`
  - Validation: only these two values.

- `export type FileNameMode = "auto" | "manual";`
  - Validation: fallback to `"auto"` on unknown persisted value.

- `export type PageId = string;`
  - Generated by ID factory (`crypto.randomUUID` fallback).

- `export interface PageDraft`
  - `id: PageId`
  - `title: string` (trimmed, non-empty fallback: `Page ${n}`)
  - `inputKind: PageInputKind`
  - `rawContent: string` (source payload)
  - `baseUrl: string | null` (resolved source URL base)
  - `previewHtml: string` (safe iframe preview document)
  - `createdAt: number` (epoch ms; supports stable ordering)

- `export interface EpubImage`
  - `id: string` (`img-${n}`)
  - `href: string` (relative within `OEBPS/images`)
  - `mediaType: string` (MIME)
  - `bytes: Uint8Array`
  - `sourceUrl: string` (absolute/data URL key)

- `export interface SanitizedHtmlResult`
  - `title: string`
  - `baseUrl: string | null`
  - `chapterBodyHtml: string`
  - `previewHtml: string`
  - `stats: { droppedNodes: number; rewrittenLinks: number; rewrittenImages: number }`

- `export interface GenerationWarning`
  - `code: "UNRESOLVED_IMAGE" | "FETCH_IMAGE_FAILED" | "MALFORMED_DATA_URL" | "EMPTY_PAGE_SKIPPED" | "UNSUPPORTED_TAG_REMOVED"`
  - `message: string`
  - `pageId?: PageId`
  - `source?: string`

- `export interface GenerationSummary`
  - `fileName: string`
  - `chapterCount: number`
  - `embeddedImageCount: number`
  - `externalImageCount: number`
  - `durationMs: number`

- `export interface BuildEpubInput`
  - `bookTitle: string`
  - `bookAuthor?: string`
  - `pages: PageDraft[]`
  - `sanitizePolicy: SanitizationPolicy`

- `export interface BuildEpubResult`
  - `blob: Blob`
  - `warnings: GenerationWarning[]`
  - `summary: GenerationSummary`

- `export interface SanitizationPolicy`
  - `dropTags: Set<string>`
  - `allowTags: Set<string>`
  - `allowExternalLinks: boolean`
  - `embedRemoteImages: boolean`
  - `maxPreviewNodes: number` (default 10)

- `export interface EpubMakerPrefs`
  - `title: string`
  - `author: string`
  - `manualFileName: string`
  - `fileNameMode: FileNameMode`
  - `sanitizeOptions: { embedRemoteImages: boolean; allowExternalLinks: boolean }`

- `export interface EpubMakerState`
  - `pages: PageDraft[]`
  - `isAdding: boolean`
  - `isGenerating: boolean`
  - `generationProgress: number | null` (0–100)
  - `showPasteFallback: boolean`
  - `pastedInput: string`
  - `warnings: GenerationWarning[]`
  - `errors: string[]`
  - `summary: string`
  - `prefs: EpubMakerPrefs`

[Testing]
Use unit-first testing for domain modules and targeted component tests for critical user flows.

Test requirements:

- `file-name.test.ts`
  - extension handling, invalid character removal, empty fallback behavior.

- `html-sanitizer.test.ts`
  - drop/allow tag behavior, link/image URL rewriting, title inference priority, preview node limit, malicious input handling.

- `page-draft.test.ts`
  - HTML/text detection, empty content rejection, deterministic title fallback.

- `epub-builder.test.ts`
  - chapter serialization, manifest/spine generation, image embedding/external fallback, warning emission, output blob creation.

- `prefs-storage.test.ts`
  - corrupt JSON recovery, unknown enum fallback, default value hydration.

- Optional component tests:
  - `EpubMetadataForm` toggles between auto/manual mode correctly.
  - `PageDraftGrid` reorder and remove actions dispatch expected handlers.

Validation strategy:

- Run tests and lint before/after migration to ensure behavior parity.
- Manually validate in browser: add from clipboard, paste fallback, remove/reorder, generate, warning handling, download filename behavior.
- Regression check for route-level rendering under `src/app/tools/layout.tsx` wrapper.

[Implementation Order]
Implement the refactor in vertical slices from core types/utilities to domain builder to UI integration, finishing with tests and verification.

1. Create feature folder skeleton and centralize all EPUBMaker types/constants.
2. Extract low-level pure utilities (`xml`, `url`, `file-name`, `id`) and add unit tests.
3. Extract domain modules (`plain-text`, `html-sanitizer`, `page-draft`, `images`) with tests.
4. Implement `epub-builder.ts` with warning/summary contracts and tests.
5. Implement service layer (`prefs-storage`, `clipboard`, `download`) and tests/mocks.
6. Build `useEpubMaker` hook to orchestrate state/actions/progress/error surfaces.
7. Split UI into focused components and compose `EpubMakerPageView`.
8. Simplify `src/app/tools/epub-maker/page.tsx` into thin feature entrypoint.
9. Add UX improvements (reorder/rename chapters, richer warnings/progress, optional sanitize toggles) through hook + UI modules.
10. Run lint/tests/manual validation and adjust edge-case handling before merge.
